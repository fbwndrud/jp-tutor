## 2. 게임별 난이도 곡선

각 게임에 4단계 난이도 티어를 적용합니다. 사용자는 직접 선택할 수 없으며, **시스템이 성과 데이터를 기반으로 자동 배정**합니다.

### 난이도 자동 배정 규칙

```
최근 3판 승률 기준:
- 90% 이상 → 다음 티어로 승격
- 70-89% → 현재 티어 유지
- 50-69% → 현재 티어 유지 (격려 메시지)
- 50% 미만 → 이전 티어로 강등
```

## 3. 스마트 추천 로직

### 3.1 추천 트리거 및 우선순위

추천은 **점수화 시스템**으로 동작합니다. 각 트리거가 후보 게임에 점수를 부여하고, 최고 점수 게임 1-2개를 추천합니다.

```
최종 추천 점수 = 기본 점수 + 컨텍스트 점수 + 신선도 점수 + 프로필 점수
```

### 3.2 트리거별 점수 배정

#### 트리거 1: 새 개념 학습 직후 (정착 강화)

**발동 조건**: 새로운 단어/문법/한자를 3개 이상 학습한 직후

| 학습 내용 | 추천 게임 | 점수 |
|-----------|-----------|------|
| 새 어휘 학습 | A. 매칭 카드 | +80 |
| 새 어휘 학습 | E. 스네이크 매치 | +70 |
| 새 어휘 학습 | L. 끝말잇기 | +50 |
| 새 문법 학습 | I. 파티클 배틀 | +80 (조사 관련 시) |
| 새 문법 학습 | D. 셔플 퍼즐 | +70 (어순 관련 시) |
| 새 한자 학습 | H. 한자 조립 | +90 |
| 새 한자 학습 | F. 따라 쓰기 | +75 |
| 새 경어 학습 | J. 경어 변환 | +90 |

**추천 멘트 예시**: "방금 배운 단어들, 매칭 카드로 한번 더 다져볼까요?"

#### 트리거 2: 복습 큐가 클 때 (복습을 재미있게)

**발동 조건**: `nextReview <= 오늘`인 개념이 10개 이상

| 복습 대상 | 추천 게임 | 점수 |
|-----------|-----------|------|
| 어휘 복습 10개+ | A. 매칭 카드 | +70 |
| 어휘 복습 10개+ | E. 스네이크 매치 | +65 |
| 어휘 복습 15개+ | L. 끝말잇기 | +60 |
| 한자 복습 5개+ | H. 한자 조립 | +75 |
| 한자 복습 5개+ | F. 따라 쓰기 | +65 |
| 문법 복습 5개+ | I. 파티클 배틀 | +70 |
| 문법 복습 5개+ | K. 문맥 추리 | +65 |

**보너스**: 복습 대상이 20개 이상이면 전체 점수 +20 (게임화 필요성 증가)

**추천 멘트 예시**: "복습할 단어가 15개나 쌓였어요! 매칭 카드로 가볍게 정리해볼까요?"

#### 트리거 3: 반복 학습 피로 감지 (다양성 제공)

**발동 조건**: 최근 3세션 연속 같은 모드(퀴즈/플래시카드)를 사용했을 때

| 조건 | 추천 게임 | 점수 |
|------|-----------|------|
| 어휘 퀴즈 3연속 | E. 스네이크 매치 | +85 |
| 어휘 퀴즈 3연속 | L. 끝말잇기 | +80 |
| 문법 퀴즈 3연속 | K. 문맥 추리 | +85 |
| 문법 퀴즈 3연속 | D. 셔플 퍼즐 | +75 |
| 한자 플래시카드 3연속 | H. 한자 조립 | +85 |
| 한자 플래시카드 3연속 | G. 듣고 그리기 | +75 |

**추천 멘트 예시**: "퀴즈 좀 많이 했죠? 기분 전환 겸 끝말잇기 한판 어때요?"

#### 트리거 4: 약점 기반 타겟팅

**발동 조건**: 특정 카테고리에 빨강(🟥) 배지가 3개 이상 존재

| 약점 유형 | 추천 게임 | 점수 |
|-----------|-----------|------|
| 조사 정답률 40% 미만 | I. 파티클 배틀 | +95 |
| 한자 읽기 정답률 40% 미만 | H. 한자 조립 | +90 |
| 한자 쓰기 정답률 40% 미만 | F. 따라 쓰기 | +90 |
| 경어 정답률 40% 미만 | J. 경어 변환 | +95 |
| 어순 오류 빈번 | D. 셔플 퍼즐 | +90 |
| 어휘 recall 낮음 | A. 매칭 카드 | +85 |
| 문맥 파악 약함 | K. 문맥 추리 | +90 |
| 청취 약함 | G. 듣고 그리기 | +85 |

**추천 멘트 예시**: "조사 부분이 좀 약한데, 파티클 배틀로 집중 훈련해볼까요?"

#### 트리거 5: 데일리 세션 게임 추천

**발동 조건**: `/jp:daily` 실행 시 세션의 마지막 활동으로 항상 1개 게임 추천

**선택 로직**:
1. 트리거 4 (약점)에 해당하는 게임이 있으면 → 해당 게임 (점수 +100)
2. 트리거 2 (복습 큐)에 해당하면 → 복습 관련 게임 (점수 +90)
3. 둘 다 아니면 → 최근 7일간 안 한 게임 중 랜덤 (점수 +70)

**추천 멘트 예시**: "오늘의 마무리 게임! 한자 조립으로 마무리해볼까요?"

### 3.3 신선도 점수 (최근 플레이 여부)

같은 게임만 계속 추천하지 않도록 감쇠 적용:

```
신선도 점수:
- 7일 이상 안 한 게임: +20
- 3-6일 전에 한 게임: +10
- 1-2일 전에 한 게임: +0
- 오늘 이미 한 게임: -30
- 오늘 2회 이상 한 게임: -60
```

### 4.2 세션 흐름 상세

```
[게임 시작]
  |
  v
[규칙 설명] ← 첫 플레이 시만 표시 (이후 "규칙 보기" 버튼으로 축소)
  |
  v
[준비 카운트다운] ← "3... 2... 1... 시작!"
  |
  v
[라운드 진행] ← 메인 게임 루프
  |
  v
[라운드 결과] ← 점수, 정답률, 소요 시간
  |
  +--→ [다음 라운드?]
  |      |
  |      +--→ "계속" → [라운드 진행]으로 복귀
  |      |
  |      +--→ "그만하기" → [세션 결과]로 이동
  |      |
  |      +--→ [최대 라운드 도달] → 자동으로 [세션 결과]로 이동
  |
  v
[세션 결과]
  ├─ 총 점수 / 정답률
  ├─ 획득 XP
  ├─ 난이도 변동 안내 ("티어 3으로 승격!")
  ├─ 틀린 문제 간단 복습 (오답 3개까지만 표시)
  └─ 연관 게임 추천 또는 학습 복귀 안내
```

### 4.3 자동 종료 조건 (플레이어 보호)

게임에 과몰입하여 본 학습을 놓치지 않도록 다음 조건에서 자동 종료를 권고합니다:

| 조건 | 행동 |
|------|------|
| 연속 15분 이상 게임 | "잠깐! 15분째 게임 중이에요. 복습할 단어가 {N}개 대기 중입니다." 안내 |

### 5.2 실패 기반 자동 연계 (Failure Routing)

특정 게임에서 실패 패턴이 나타나면 보완 게임을 자동 추천합니다.

| 게임 | 실패 패턴 | 추천 보완 게임 | 추천 메시지 |
|------|-----------|---------------|-------------|
| G. 듣고 그리기 | 글자 형태 유사도 50% 미만 반복 | F. 따라 쓰기 | "글자 모양이 좀 아쉬워요. 따라 쓰기로 먼저 연습해볼까요?" |
| E. 스네이크 매치 | 같은 쌍을 반복해서 틀림 | A. 매칭 카드 | "이 단어들 좀 헷갈리죠? 매칭 카드로 먼저 익혀봐요." |
| L. 끝말잇기 | 5턴 이내 탈락 3회 연속 | A. 매칭 카드 / E. 스네이크 매치 | "어휘를 좀 더 쌓으면 끝말잇기가 훨씬 재밌어져요!" |
| K. 문맥 추리 | 조사 관련 빈칸 오답 반복 | I. 파티클 배틀 | "조사 부분이 좀 약한 것 같아요. 파티클 배틀로 훈련해볼까요?" |
| K. 문맥 추리 | 경어 관련 빈칸 오답 | J. 경어 변환 | "경어 변환 연습을 먼저 하면 문맥 추리가 훨씬 쉬워질 거예요." |
| D. 셔플 퍼즐 | 어순 배열 실패 반복 | I. 파티클 배틀 | "조사를 좀 더 익히면 어순이 자연스럽게 잡혀요!" |
| H. 한자 조립 | 부수 위치 오류 반복 | F. 따라 쓰기 | "한자 구조를 눈으로 익히면 조립이 쉬워져요. 따라 쓰기 해볼까요?" |
| I. 파티클 배틀 | 티어3-4에서 문맥 파악 실패 | K. 문맥 추리 | "문맥 읽기를 연습하면 조사 선택이 더 정확해져요!" |

### 5.3 성공 기반 상위 게임 추천 (Success Routing)

특정 게임에서 높은 성과를 보이면 상위 게임으로 유도합니다.

| 게임 | 성공 조건 | 추천 상위 게임 | 추천 메시지 |
|------|-----------|---------------|-------------|
| A. 매칭 카드 | 티어3 이상에서 3연승 | E. 스네이크 매치 | "매칭 카드 고수시네요! 스네이크 매치에서 실력을 테스트해보세요." |
| F. 따라 쓰기 | 티어3 빈 캔버스 모드 성공률 80%+ | G. 듣고 그리기 | "이제 눈 감고도 쓰겠는데요? 소리만 듣고 써보는 건 어때요?" |
| I. 파티클 배틀 | 티어3 이상 정답률 85%+ | K. 문맥 추리 | "조사 실력 최고! 이제 전체 문맥을 읽는 연습을 해볼까요?" |
| A/E (어휘 그룹) | 두 게임 모두 티어2 이상 | L. 끝말잇기 | "단어 실력이 많이 늘었네요! 끝말잇기로 실전 테스트!" |
| H. 한자 조립 | 티어3에서 체인 5연속 | F. 따라 쓰기 (한자) | "한자 구조를 잘 알고 있으니, 직접 써보는 건 어때요?" |

### 5.4 데일리 세션 내 게임 배치

`/jp:daily` 세션에서의 게임 추천 위치:

```
[데일리 세션 흐름]

1. 출석 체크 + 스트릭 업데이트
2. 대시보드 요약
3. 복습 세션 (에빙하우스 기반)
   └─ 복습 대상 10개+ → 게임 추천 삽입 가능 (선택사항)
4. 새 개념 학습
   └─ 학습 후 → 게임 추천 삽입 (정착 강화)
5. ★ 마무리 미니게임 (1라운드) ← 항상 추천
6. 세션 종료 + 결과 요약
```

**마무리 미니게임 선택 우선순위**:
1. 오늘 학습한 내용과 가장 관련 높은 게임
2. 약점 보강 게임
3. 7일 이상 안 한 게임
4. 랜덤

```json
    "daily_game_played": false,
```

### 해금 체크 함수 (의사코드)

```
function getUnlockedGames(level, profile, minigameStats):
  unlocked = []

  // Tier 1: 항상 해금 (레벨 조건만)
  if level >= "Pre-N5":
    unlocked += ["matching_card", "shuffle_puzzle", "stroke_practice"]
  if level >= "N5":
    unlocked += ["snake_match", "radical_assembly", "particle_battle", "shiritori"]
  if level >= "N4":
    unlocked += ["keigo_transformer", "context_detective"]

  // 조건부 해금
  if level >= "N5" AND strokePractice.hiragana_passed >= 20:
    unlocked += ["listen_draw"]

  return unlocked
```

### 추천 점수 계산 함수 (의사코드)

```
function recommendGame(context):
  candidates = getUnlockedGames(...)
  scores = {}

  for game in candidates:
    score = 0

    // 트리거 1: 새 개념 학습 직후
    if context.just_learned_new:
      score += getTrigger1Score(game, context.learned_type)

    // 트리거 2: 복습 큐
    if context.review_queue_size >= 10:
      score += getTrigger2Score(game, context.review_categories)

    // 트리거 3: 반복 피로
    if context.same_mode_streak >= 3:
      score += getTrigger3Score(game, context.recent_mode)

    // 트리거 4: 약점 기반
    if context.red_badges >= 3:
      score += getTrigger4Score(game, context.weak_categories)

    // 트리거 5: 데일리 마무리
    if context.is_daily_ending:
      score += 100 if game == getBestDailyGame(context) else 0

    // 신선도
    score += getFreshnessScore(game, minigameStats)

    // 프로필 보너스
    score += getProfileBonus(game, profile)

    scores[game] = score

  return top2(scores)
```
