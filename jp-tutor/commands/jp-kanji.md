# 한자 학습 (jp:kanji)

## 사전 확인

1. `jp-data/progress.json` 파일을 읽습니다.
2. 파일이 없으면 "아직 학습 프로필이 없습니다. `/jp:start`로 먼저 초기 설정을 해주세요."라고 안내하고 종료합니다.

## 레벨 및 그룹 결정

- **레벨**: 인자로 지정되면 해당 레벨, 없으면 `progress.json`의 `level` 값 사용.
- **그룹**: 인자로 지정되면 해당 그룹 (예: "음식", "동작", "자연"), 없으면 미학습/취약 한자 우선 선택.

## 한자 선택

- 한 세션에 **5~8개** 한자를 선택합니다 (과부하 방지).
- 우선순위: 미학습 > 취약(🟥) > 복습 시기 도래 순.

## 플래시카드 학습

각 한자에 대해 다음 정보를 카드 형식으로 제시합니다:

```
━━━━━━━━━━━━━━
    食
━━━━━━━━━━━━━━
음독: ショク, ジキ
훈독: た(べる), く(う)
의미: 먹다, 음식
부수: 食 (식)
획수: 9획
관련 단어: 食事(しょくじ), 食べ物(たべもの), 和食(わしょく)
유사 한자: 飲, 飯, 餅
━━━━━━━━━━━━━━
```

- 모든 한자를 순서대로 보여줍니다.
- 학습자가 충분히 확인한 후 퀴즈로 넘어갑니다.

## 플래시카드 HTML 생성 (선택)

학습한 한자를 플래시카드로 저장할지 물어봅니다: "플래시카드 HTML을 생성할까요? (Y/N)"

Y를 선택하면:
- `templates/flashcard.html` 템플릿을 참조하여 생성합니다.
- 파일은 `jp-data/flashcards/kanji-{날짜}-{그룹}.html`에 저장합니다.

## 확인 퀴즈

`skills/jp/references/quiz-rules.md` 규칙을 따릅니다.

### 구성
- **4문제**, 4지선다
- 학습한 한자 범위 내에서 출제
- 문제 유형을 혼합합니다:
  - 한자 -> 읽기: 한자를 보고 올바른 읽기 고르기
  - 한자 -> 의미: 한자를 보고 올바른 의미 고르기
  - 의미 -> 한자: 의미를 보고 올바른 한자 고르기
  - 읽기와 의미를 균형 있게 섞습니다

### 선택지 생성
- 유사한 모양, 같은 부수, 비슷한 음독에서 오답을 선택합니다.
- 선택지 형식을 통일합니다.

### 무힌트 정책
퀴즈 중 힌트는 절대 제공하지 않습니다.

### 즉시 피드백
각 문제 답변 직후 정답/오답 표시 및 해설을 제공합니다.

## 라운드 종료 후

### 결과 요약
```
🈁 한자 학습 결과
━━━━━━━━━━━━━━
학습 한자: {한자 목록}
퀴즈 점수: {correct}/4
획득 XP: +{xp}
━━━━━━━━━━━━━━
```

### progress.json 업데이트
- `kanji.concepts`에서 각 한자를 **읽기**와 **의미** 별도 추적합니다:
  - 키 형식: `{한자}_reading`, `{한자}_meaning`
  - 정답: `correct` +1, `attempts` +1
  - 오답: `attempts` +1
  - `lastReview`, `nextReview` 에빙하우스 간격에 따라 갱신
- XP 부여: 플래시카드 학습 +10, 퀴즈 정답 +10, 오답 +2
- 스트릭 업데이트

### 오답 복습
틀린 한자가 있으면 세션 마지막에 해당 한자 카드를 한 번 더 보여줍니다.

## 언어 규칙
- 모든 설명과 피드백은 **한국어**로 제공합니다.
- N4 이하 학습자에게는 관련 단어에 반드시 **후리가나**를 병기합니다.
