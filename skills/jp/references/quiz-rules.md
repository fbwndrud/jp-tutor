# 퀴즈 생성 규칙

이 문서는 모든 퀴즈 유형(어휘, 문법, 한자, 독해 등)에 적용되는 공통 규칙을 정의합니다.

## 기본 라운드 구성

- **4문제 x 4선택지** 1라운드
- 사용자가 원하면 라운드 수 조절 가능
- 각 문제 후 즉시 정답/해설 제공 (누적 채점 아님)

## Zero-Hint Policy

선택지 작성 시 정답에 유리한 단서를 절대 포함하지 않습니다.

금지 패턴:
- 정답만 구체적이고 오답은 모호한 경우
- 정답만 문법적으로 자연스러운 경우
- 정답의 길이가 유독 다른 경우
- 선택지 설명에 "~이므로", "~때문에" 같은 정답 암시

### 예시 — 나쁜 선택지

```
Q: 「食べる」의 て형은?
A) 食べて  ← 정답 (유일하게 올바른 형태)
B) 食べって  ← 존재하지 않는 형태라 즉시 제외됨
C) 食べいて  ← 비문법적
D) 食べりて  ← 비문법적
```

### 예시 — 좋은 선택지

```
Q: 「食べる」의 て형은?
A) 食べて  ← 정답
B) 食って  ← 食う의 て형 (실제 존재하는 형태)
C) 食べって  ← 5단 활용 혼동 (흔한 오류)
D) 食べれて  ← 가능형 혼동 (흔한 오류)
```

오답은 학습자가 실제로 혼동하기 쉬운 형태여야 합니다.

## 그럴듯한 오답 생성 규칙

### 어휘 퀴즈
- 같은 JLPT 레벨의 단어를 오답에 배치
- 의미가 비슷하거나 글자가 비슷한 단어 우선 (예: 経済/経営, 以上/以下)
- 같은 품사의 단어만 선택지에 포함

### 문법 퀴즈
- 비슷한 문형을 오답에 배치 (예: ~ても/~てから/~たら)
- 같은 접속 형태의 문법끼리 묶기
- 학습자의 error_notes에 기록된 혼동 패턴 활용

### 한자 퀴즈
- 모양이 비슷한 한자를 오답에 (예: 持/待/特)
- 같은 부수 또는 같은 음의 한자
- 음독/훈독 혼동 유도 (흔한 실수)

## 반복 금지

같은 개념을 재출제할 때:
- 이전과 동일한 문제문 사용 금지
- 새로운 예문/맥락에서 출제
- progress.json의 해당 개념 error_notes를 참고하여 약점을 타겟팅

## 난이도 조절

### 진단 퀴즈 (셋업 시)
- 쉬움 40% / 보통 40% / 어려움 20%
- 해당 레벨의 핵심 개념 위주
- 빠르게 레벨을 파악하는 것이 목적

### 드릴 (약점 보강)
- 보통 30% / 어려움 70%
- 🟥 배지 개념을 집중 출제
- error_notes 기반 오류 패턴 반복 훈련

### 복습 (정기 복습)
- 쉬움 20% / 보통 60% / 어려움 20%
- 최근 학습한 개념 + 시간 경과된 🟩 개념
- 망각 곡선 고려: 오래된 개념 우선

### 일반 학습
- 쉬움 30% / 보통 50% / 어려움 20%
- 현재 레벨의 미측정(⬜) 개념 위주

## 퀴즈 타입별 규칙

### 어휘
1. **뜻 맞추기**: 일본어 단어 → 한국어 뜻 (4선택지)
2. **예문 빈칸**: 예문에서 단어 빈칸 → 적절한 단어 선택
3. **유의어 구분**: 비슷한 뜻의 단어 중 맥락에 맞는 것 선택

### 문법
1. **올바른 형태 선택**: 주어진 문장에서 올바른 문법 형태 선택
2. **오류 찾기**: 4개 문장 중 문법 오류가 있는 문장 찾기
3. **문장 완성**: 문장의 빈칸에 적절한 문법 표현 선택

### 한자
1. **읽기**: 한자 → 히라가나 읽기 선택 (음독/훈독 구분)
2. **뜻**: 한자 → 한국어 뜻 선택
3. **한자 선택**: 히라가나 → 올바른 한자 선택

## 채점 후 진도 업데이트

퀴즈 완료 후 반드시:
1. 각 문제의 개념별로 progress.json의 attempts/correct 업데이트
2. 배지 재계산
3. 오답 시 error_notes에 오류 패턴 추가 (단순 "틀림"이 아닌 구체적 패턴)
4. last_tested 갱신
5. session_log에 퀴즈 결과 요약 추가

### 오류 패턴 기록 예시

```json
{
  "error_notes": [
    "受身形과 使役形을 혼동",
    "음독 セイ를 ショウ로 읽음",
    "~ている(진행)과 ~てある(결과상태) 구분 실패"
  ]
}
```
